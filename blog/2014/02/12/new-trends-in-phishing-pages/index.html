New Trends in Phishing Pages?
Feb 12th, 2014

What I’m describing in this post is no rocket science. It’s neither a state of the art analysis. It’s an observation that I made in how a technique, used in obfuscating certain malicious web code is now also been used in obfuscating phishing pages.

Today, via via, I received a phishing mail from someone claiming to be Apple:

First, I retrieved what’s on the URL via wget:

Nothing spectacular here. The URL mentioned in the mail redirects to a file in another directory on the same web server. Here, an index.html is offered.

When looking at the index.html file, you notice that this isn’t regular JavaScript syntax. As you can see, it’s clearly obfuscated.

The perpetrators left a useful comment behind:

As you can see, the obfuscated code above is no encryption at all but just some strings that are represented in hex code.

When you deobfuscate the code you get the real HTML (and mainly JavaScript) code:

This looks far more like a phishing page!

The included jquery-2.1.0.js file contained following comment:

From the date mentioned in this comment, we can speculate that the phishing sitemight not be older then 2.5 weeks. A ‘diff’ with the JQuery 2.1.0 file for the official website shows that this file is not modified.

As you can see also the <article> tag is used. As this tag is introduced first in HTML5, this phishing site seems to be compatible with it :).

A malicious page that is largely built from obfuscated JavaScript: it is not something new. However, applying this technique to usual phishing sites, is rather uncommon, but not completely illogical. This kind of phishing site obfuscation will probably occur more and more in the future.
